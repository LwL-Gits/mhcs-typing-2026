🌐 How to Host on mhcs-nj.org — Step by Step

There are three realistic paths depending on what access you have. Start with Option A.

Option A — Email the file to your web host/IT (Easiest, ~5 minutes your time)

Most school websites are managed by a web company or IT person. This is likely true for mhcs-nj.org.

  1. Email your web administrator (whoever manages mhcs-nj.org) at their support address — or reply to whoever set up your languagedean@mhcs-nj.org email.
  2. Attach mhcs_typing_contest.html and ask them to upload it so it's accessible at a URL like: https://mhcs-nj.org/typing/ or https://mhcs-nj.org/typing-contest.html
  3. Ask them to do it before May 14 at 8 AM. It's a single file — a 2-minute upload for them.
  4. Share that URL with students.

This is the recommended path if you have any IT support contact.

Option B — GitHub Pages (Free, permanent, no cost, ~15 minutes)

This is the best self-service option. GitHub Pages hosts static HTML files for free with a real URL.

Step 1 — Create a free GitHub account

  • Go to github.com → click Sign up
  • Use any email (your languagedean@mhcs-nj.org works fine)

Step 2 — Create a new repository

  • Click the + (top right) → New repository
  • Name it: mhcs-typing-2026
  • Set it to Public
  • Check "Add a README file"
  • Click Create repository

Step 3 — Upload the HTML file

  • In your new repository, click Add file → Upload files
  • Drag and drop mhcs_typing_contest.html onto the page
  • Important: rename it to index.html before uploading (just rename it on your computer first)
  • Scroll down, click Commit changes

Step 4 — Enable GitHub Pages

  • Click Settings (tab at top of repository)
  • Scroll down to Pages in the left sidebar
  • Under Source, select Deploy from a branch
  • Set Branch to main, folder to / (root)
  • Click Save

Step 5 — Get your URL

  • After 1–2 minutes, your site will be live at: https://[your-github-username].github.io/mhcs-typing-2026/
  • Example: https://languagedean-mhcs.github.io/mhcs-typing-2026/

Share this URL with students — it works on any device, anywhere.

Option C — Netlify Drop (Fastest of all, ~2 minutes, free)

No account needed for the basic version.

  1. Go to app.netlify.com/drop in your browser
  2. Rename mhcs_typing_contest.html to index.html on your computer first
  3. Drag and drop the file onto the Netlify Drop page
  4. Netlify instantly gives you a URL like https://random-name-12345.netlify.app
  5. You can optionally create a free Netlify account to get a cleaner URL like https://mhcs-typing-2026.netlify.app

⚠️ One Important Note About Score Saving

The standalone HTML file saves scores in localStorage — meaning scores are saved per browser/device. This is different from the Claude chat version, which used shared cloud storage.

Practical impact: Students will see their own history and the leaderboard only updates when they're on the same device, OR if you want true shared leaderboard across devices, I can upgrade the file to sync scores via a free service like JSONBin or Firebase (a 30-minute addition). Just say the word and I'll add that.

For a one-week school contest, many schools simply have students use one consistent device (school computer or their own laptop) throughout, which works fine with localStorage.
